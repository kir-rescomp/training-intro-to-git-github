1. What is Version Control?

2. What is Git?

Understanding the Git Workflow

  • Working Directory: Where you edit and modify your files
  • Staging Area (Index): A preparation zone where you select which changes to include in the next commit
  • Local Repository: Where Git permanently stores committed snapshots of your project

3. How GitHub Works

Key GitHub Concepts

  • git clone: Download a complete copy of a repository from GitHub to your computer
  • git push: Upload your local commits to the remote repository on GitHub
  • git pull: Download changes from GitHub and merge them into your local branch
  • git fetch: Download changes from GitHub but don't merge them (safer than pull)
  • Branches: Separate lines of development that allow you to work on features without affecting the main code
  • Issues: Track bugs, enhancements, and tasks
  • Pull Requests: Propose changes and facilitate code review before merging
